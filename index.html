INTRODUCTION

The Flexible Layout (flexible box) model began development sometime before 2008, with the aim of being able to quickly and easily organize responsive elements on a webpage. Flexbox rolled fully in 2009, and has seen numerous revision in the fifteen years since it's introduction.

Flexbox has a one-dimensional layout whereby users can organize their elements along a row or a column axis. Also included in the flex layout is the ability for the container to alter it's items' width and height. A flex container automatically expands items within it to fill available free space, or shrinks them to prevent oveflow.

The flexbox layout is most apt for components of an application, as well as smaller-scale layouts. For larger projects or whole-site layouts, we recommend reading into CSS Grid

WHAT YOU SHOULD KNOW

Before you begin delving into learning the basics of the Flexbox module, you should have a functional understanding of basic HTML and the other forms of positioning elements such as:

  • The CSS float property and it's attribute/
  • Centering elements by using the CSS margin property.
  • Positioning text with the CSS text-align property.
DISPLAY ATTRIBUTES

CSS flexbox is only one of several attributes for the display property. Some display properties set the behavor of the element itself in the layout, and others (including flexbox) change how the children of an element are positioned. The display attributes that change the element's behavior itself are:

  • The block attribute, which generates the element on a block level, creating line breaks before and after the element
  • The inline attribute, which doesn't generate breaks, and treats the element as if it should be one the same line (provided there is space)
  • And, confusingly, the inline-block attribute. Inline-block does not generate line-breaks before and after the element, but lets you set a width and height for the element, as well as maintain respect for the elements defined margin and padding

When written as code, these attributes look like:

display: block;
/* or */
display: inline;
/* or */
display: inline-block;

The display attributes that alter the way it's children are laid out are:

  • The flex attribute
  • The grid attribute

There is actually one more display attribute that is relevant here:

display: inline-flex;

By default the flex container behaves as a block-level element. By setting it to inline-flex, it makes the container behave as an inline element. There are use-cases for this, so keep it in the back of your mind while coding.
